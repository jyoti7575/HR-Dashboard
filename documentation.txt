| `birthdate` | String, mixed formats         |
| `hire_date` | String                        |
| `termdate`  | Partial nulls + timezone      |
| All others  | Object instead of category    |
c. HR Red Flags:
	termdate exists ‚Üí attrition analysis
	hire_date + birthdate ‚Üí age at hire, tenure
	department, jobtitle, location ‚Üí org design
	gender, race ‚Üí DEI analytics

2. DATA CLEANING
a. Removed rows which are having invalid ID:
	data = data[data['id'].notna()].copy()
b. Standardizing date columns:
	df['birthdate'] = pd.to_datetime(df['birthdate'], errors='coerce')
	df['hire_date'] = pd.to_datetime(df['hire_date'], errors='coerce')

	df['termdate'] = (
    		pd.to_datetime(df['termdate'], errors='coerce')
      			.dt.tz_localize(None)
)
c. Fix Categorical Columns:
	cat_cols = [ 'gender', 'race', 'department', 'jobtitle', 'location', 'location_city', 'location_state']
	for col in cat_cols:
    		data[col] = data[col].astype('category')

3. FEATURE ENGINEERING:
a. age: 
	today = pd.Timestamp.today()
	data['age'] = (today - data['birthdate']).dt.days // 365
b. tenure: 
	data['tenure_years'] = ((data['termdate'].fillna(today) - data['hire_date']) .dt.days / 365).round(1)
c. employee status:
	data['employment_status'] = data['termdate'].apply( lambda x: 'Terminated' if pd.notna(x) else 'Active')

4. FACT TABLE:
The FactEmployees table represents the core transactional entity of the HR system at the employee grain (one row per employee). It stores measurable attributes and key dates required for workforce analytics.

5. DIM TABLE:
a. DimDepartment: DimDepartment provides a normalized list of organizational departments, used to categorize employees by functional area.
b. DimLocation: DimLocation captures the geographic structure of the workforce, including location, city, and state.
c. DimDate (Calendar Table): DimDate is a standardized calendar dimension used to perform accurate time-based analysis across hiring and termination events.

6. WHY STAR SCHEMA?
- Ensures a clean separation of metrics (facts) and descriptive attributes (dimensions)
- Improves dashboard performance, scalability, and maintainability
- Aligns with enterprise data warehousing best practices

9. DAX:
a. Headcount =
CALCULATE(
    COUNT(Employees[id]),
    Employees[employment_status] = "Active"
)
b. Attrition =
CALCULATE(
    COUNT(Employees[id]),
    Employees[employment_status] = "Terminated"
)
c. Attrition Rate =
DIVIDE([Attrition], [Attrition] + [Headcount])
d. Avg Tenure =
AVERAGE(Employees[tenure_years])
e. Average Age = AVERAGE(FactEmployees[age])
f. Attrition Risk Flag = 
IF(
    [Attrition Rate] > 0.15,
    1,
    0
)
g. Attrition Color = 
IF(
    [Attrition Risk Flag] = 1,
    "#C62828",   -- üî¥ High Risk Red
    "#81C784"    -- üü¢ Normal Green
)
h. Departments at Risk = 
CALCULATE(
    DISTINCTCOUNT(DimDepartment[department]),
    FILTER(
        ALLSELECTED(DimDepartment[department]),
        [Attrition Risk Flag] = 1
    )
)
i. Workforce Risk Level = 
VAR AttritionRate = [Attrition Rate]
VAR AvgTenure = [Avg Tenure]

RETURN
SWITCH(
    TRUE(),

    AttritionRate > 0.20,
    "üî¥ High Risk",

    AttritionRate > 0.10 || AvgTenure < 3,
    "üü† Watch",

    "üü¢ Stable"
)
j. Dept Recommendation = 
VAR DeptAttrition = [Attrition Rate]

RETURN
SWITCH(
    TRUE(),

    DeptAttrition > 0.20,
    "üö® Attrition in this department is critically high. Immediate retention action required.",

    DeptAttrition > 0.15,
    "‚ö†Ô∏è Attrition exceeds acceptable threshold. Review exits, workload, and manager effectiveness.",

    "‚úÖ Attrition in this department is within acceptable limits."
)
k. Executive Recommendation = 
VAR AttritionRate = [Attrition Rate]
VAR AvgTenure = [Avg Tenure]

RETURN
SWITCH(
    TRUE(),

    AttritionRate > 0.20 && AvgTenure < 2,
    "üö® High attrition driven by early-tenure exits. Strengthen onboarding, role clarity, and first-year engagement programs.",

    AttritionRate > 0.20,
    "‚ö†Ô∏è Attrition exceeds acceptable threshold. Conduct department-level exit analysis and manager effectiveness review.",

    AvgTenure < 2,
    "‚ö†Ô∏è Low average tenure indicates possible role-fit or hiring quality issues. Review recruitment and probation processes.",

    AttritionRate <= 0.10 && AvgTenure >= 4,
    "‚úÖ Workforce is stable with strong retention. Focus on leadership development and internal mobility.",

    "‚ÑπÔ∏è Workforce metrics are within normal range. Continue monitoring retention and engagement trends."
)
l. Total Employees :=
CALCULATE(
    COUNT(FactEmployees[id]),
    FactEmployees[employment_status] = "Active"
)
m. Departments Covered :=
DISTINCTCOUNT(DimDepartment[department])
n. States Covered :=
DISTINCTCOUNT(DimLocation[location_state])
o. Employees Under 30 % :=
DIVIDE(
    CALCULATE(
        COUNT(FactEmployees[id]),
        FactEmployees[age] < 30,
        FactEmployees[employment_status] = "Active"
    ),
    [Total Employees]
)
p. Age Group Sort =
SWITCH(
    TRUE(),
    FactEmployees[age] < 30, 1,
    FactEmployees[age] >= 30 && FactEmployees[age] <= 45, 2,
    3
)
q. Early Tenure Attrition Count :=
CALCULATE(
    COUNT(FactEmployees[id]),
    FactEmployees[employment_status] = "Terminated",
    FactEmployees[tenure_years] <= 2
)
r. Early Tenure Attrition % :=
DIVIDE(
    [Early Tenure Attrition Count],
    [Attrition]
)
